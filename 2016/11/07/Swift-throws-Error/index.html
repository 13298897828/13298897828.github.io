Swift throws Error

发表于 2016-11-07   |   分类于 swift

对于swift 2.0 中的引入了新的异常处理机制

基本用法:

自定义错误类型

enum MyError: Error {
    case ErrorTypeA
    case ErrorTypeB
}

无返回值

func testMethod() throws   {
    guard let xx = xxx else {
    throw ErrorTypeA
}

}

有返回值

func testMethod() throws -> String  {
    guard let xx = xxx else {
    throw ErrorTypeA
    }
}

调用方法

do {

    let str =  try testMethod()
    }catch let error {
    print(error)
}

抛出异常后 testMethod后面的部分不会执行

func testMethod() throws  {
    defer {
        print("step 1")
    }
    print("step 2")
    let b = "2"
    guard let xx = xxx else {
        throw ErrorTypeA
    }
    print("step 3")
}
do {
    let str =  try testMethod()
    }catch let error {
        print(error)
}

以上代码输出的顺序为
当抛出异常时: step 2 , 1 , error
不抛出异常时: step 2 , 3 , 1
