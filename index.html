关于swift中的泛型

发表于 2016-06-07   |   分类于 swift

记录下对swift中泛型的认识过程

刚开始对泛型不了解,不了解这东西什么作用,也就忽视了,随着编程中的需要,无意中了解到其神奇之处,感觉原来真的是暴殄天物,罪过,罪过,简单写下简单的使用.
<!——more——>

泛型是可以根据你的需要来定制的数据类型,可以提高代码重用,并且可以让你的代码更加清晰,明了.

举个极客学院看到的例子,在一个数组中寻找自己自己想要的元素下标:

let someArray = ["qq","ee","aa","ww"]
    func findIndex(arr:[String],valuetoFind:String)->Int? {
    for (index,value) in arr.enumerated()  {
        if value == valuetoFind {
        return index
        }
    }
    return nil
}
findIndex(arr: someArray, valuetoFind: "ee")

感觉还不错,用着很方便,小小的成就感.但是如果需求变了,数组类型由String变成Int,难道要重新写一遍吗.内心是拒绝的~,copy一份改一改吧. 等等,不能这样!!!

先说下泛型的使用
fund someFunc(type:SomeType) ->SomeType{
return type
}
someFunc(type: 2)
someFunc(type:”str”)

someType就是我们定义的泛型,类似上面的调用,可以看到参数可以使Int,String,对象等等,让我们的函数更加灵活,上面的比较方法就可以改写成以下方式:
func findSomething(array:[T],valueTofind:T)->Int? {
for (index,value) in array.enumerated() {
if value == valueTofind {
return index
}
}
return nil
}
我们利用以上函数就可以进行,Int,String等类型的查找
但是以上函数调用时会发现编译报错

google了一下,是这么解释的,不是所有类型都是可以比较相等的~~
所以要给泛型加条件 类似的还有 Comparable
就可以编译通过了.
这就涉及到了另一个问题,给泛型加条件,我们想要一个符合我们需求的类型作为我们的函数参数或返回值.
这时候我们就要添加类型约束,以数组为例:
protocol Container {
associatedtype ItemType
mutating func append( item: ItemType)
var count: Int { get }
subscript(i: Int) -> ItemType {
get
}
}
func allItemsMatch
( someContainer: C1, _ anotherContainer: C2) -> Bool
where C1.ItemType == C2.ItemType, C1.ItemType: Equatable {

// 检查两个容器含有相同数量的元素
if someContainer.count != anotherContainer.count {
    return false
}

// 检查每一对元素是否相等
for i in 0..<someContainer.count {
    if someContainer[i] != anotherContainer[i] {
        return false
    }
}

// 所有元素都匹配，返回 true
return true

}
