swift 制作 Framework 静态?动态?

发表于 2016-11-02   |   分类于 swift

由于现在所写项目用’Swift’开发, 苹果要求使用’Swift’写的库,必须通过动态链接引入

此处好多坑,网上查到的好多都在说什么苹果不支持动态库,审核会被拒… 那是iOS8之前的事情了.swift出来之后就没有了好吗
以下为此观点支持理由:
(swift项目在 ‘Cocopods’ 内引入第三方库 需要在Podfile顶部加上”use_frameworks”,之后pod的库都会被编译成动态链接库,而以fake framework引入的pod仍然会编译到主App里面。动态库都放在App里面的Frameworks目录，Swift相关的动态库也都拷贝进来了，所以支持Swift会导致包变大。)
如果不添加use_framework会’Cocopods’导入会报以下错误
Mach-O dynamically linked shared library arm
Mach-O 64-bit dynamically linked shared library
因为fake framework和源代码pod分别会编译成静态库和动态库，这样会导致一个问题，就是如果源码pod又依赖fake framework，那就没办法了。CocoaPods发现这种情况会提示下面这个错误。
target has transitive dependencies that include static binaries: (xxx.framework, xxx.framework)
pod install时会把静态库编译到App里面，源码编译成的动态库没法依赖它。最终的解决方案只能是CocoaPods对fake framework和源码pod一视同仁，都编译成动态库，这样彼此才能依赖。不知道CocoaPods什么时候会支持这样。

Swift中利用SwiftyJson和ObjectMapper解析数据

发表于 2016-10-15   |   分类于 swift

1.在框架中到入swiftyJson,和objectMapper.

本人是用Cocopods到入的
pod ‘SwiftyJSON’
pod ‘ObjectMapper’
如果使用’Alamofire’可以换用’AlamofireObjectMapper’
如果框架中有多个数据解析框架的 可以像我一样 使用’SwiftyJSON’和’ObjectMapper’

2.利用objectMapper创建model

import ObjectMapper
class Man: Mappable {

var name: String?

var age: Int = -1

var married: Bool = false

required init?(map: Map) {

}

func mapping(map: Map) {

name <- map[“name”]
age <- map[“age”]
married <- map[“married”]

}
}
以上是一个简单model的创建过程,需要注意的是要实现下面的两个方法.需要 ‘xxx <- map[“xxx”]’ 对每个字段进行转换.

3.关于多层model嵌套的处理

  • 1 这个是外层的model

import ObjectMapper
// modelPr
class PrModel: Mappable {

var info : [[[AnyObject]]]!

var name : String!

var prod : [ProdDetailModel]!

var step : Int!

required init?(map: Map) {

}

func mapping(map: Map) {

info <- map[“info”]

prod <- map[“modelProdDetailModel”]

name <- map[“name”]

step <- map[“step”]

}
}

  • 2 这个是在上面model中嵌套的model

// modelProdDetailModel
class ProdDetailModel:Mappable {

var img : String!

var name : String!

var tip : [String]!

required init?(map: Map) {

}

func mapping(map: Map) {

img <- map[“img”]

name <- map[“name”]

tip <- map[“tip”]

}
}
第二个model ‘ProdDetailModel’的实现方式是没有不同的,
第一个model嵌套了第二个model 需要注意的是要在定义class:Mapper 实现第二个model的protocol,定义变量和实现mapping方法时也需要实现.
多层model原理相同原理

json转model

单层 model

如果 当数据格式为字典 获取 “man”字段下的数据
let json = JSON(data: response.data)
let model = Mapper().map(JSON: json[“man”])

多层model

对于model嵌套解析
let json = JSON(data: response.data)
let model = Mapper>().map(JSON: json.rawValue as! [String : Any])
对于多层model解析需要注意的其实就是Mapper<>内所需填写的就是外层model的格式,Mapper<>.map(JSON:)函数只是其中一种解析方式还有对 String,Dictionary 数据的处理.这个就要大家自己去摸索了,本人只是把比较常见的解析方式列举出来.
